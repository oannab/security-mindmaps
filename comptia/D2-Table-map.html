Cryptographic Attacks
vulnerability
Weak/Outdated Cipher Negotiation

System accepts legacy protocols (SSL, RC4, DES) during handshake

HIGH
attack
Downgrade Attack

MITM forces negotiation to weakest supported cipher; then cracks weak encryption

indicators
  • TLS version downgrade in handshake logs
  • Unexpected cipher suite negotiated
  • On-path network anomalies
mitigation
  • Disable SSL, TLS 1.0/1.1, RC4, DES
  • Enforce TLS 1.2/1.3 minimum
  • Strong cipher suite configuration
  • HSTS (HTTP Strict Transport Security)
web server / load balancer / VPN
vulnerability
Weak Hashing (MD5, SHA-1) / No Salting

Collision-vulnerable hash algorithms; unsalted hashes enable rainbow table attacks

HIGH
attack
Collision / Birthday / Password Cracking

Birthday: find two inputs with same hash. Rainbow table: precomputed hash lookup to crack unsalted passwords

indicators
  • Credential stuffing attempts
  • Mass authentication failures then success
  • Leaked password database matched
mitigation
  • Use SHA-256/SHA-3, bcrypt, Argon2
  • Salting all passwords
  • Key stretching (PBKDF2)
  • MFA (credential alone insufficient)
authentication system / DB / IAM
Password / Credential Attacks
vulnerability
Weak Password Policy / No Lockout

Short, simple, or reused passwords; no account lockout threshold

HIGH
attack
Brute Force

Systematically tries every possible password combination against an account or hash

indicators
  • Account lockout events
  • High volume failed auth attempts (same account)
  • Out-of-cycle login attempts
mitigation
  • Account lockout policy
  • Long complex passphrases
  • MFA
  • CAPTCHA on login forms
  • Rate limiting auth endpoints
IAM / AD / web app / VPN
vulnerability
Password Reuse / Common Passwords

Users reuse passwords across services; common passwords not blocked

HIGH
attack
Password Spraying

Tests one common password (e.g. "Summer2024!") across many accounts — evades lockout policies

indicators
  • Failed auth spread across many usernames
  • Slow, low-volume pattern across accounts
  • Impossible travel after successful login
mitigation
  • MFA (critical — spray succeeds with just password)
  • Password managers
  • UEBA / behavioral analytics
  • Block known common passwords
  • Passwordless authentication
IAM / AD / cloud SSO / SIEM

Zero-Day Vulnerabilities
vulnerability
Zero-Day

Unknown vulnerability — no patch exists; vendor may be unaware. Often chained with other exploits

CRITICAL
attack
APT / Nation-State Exploitation

Zero-days command high value; used by sophisticated actors, sold on dark web, weaponized before disclosure

indicators
  • No matching CVE for observed behavior
  • Anomalous resource consumption
  • Unknown process / blocked content
  • Threat intel feed alerts
  • Missing logs (attacker covering tracks)
mitigation
  • Defense-in-depth (no single point)
  • EDR / XDR behavioral detection
  • Sandboxing unknown executables
  • Least privilege (limits blast radius)
  • Network segmentation
  • Threat hunting & OSINT feeds
  • Rapid patching once CVE published
all platforms — defense-in-depth required
Physical Attacks
vulnerability
Weak Physical Access Controls

No badge readers, unsecured server room, unattended workstations

HIGH
attack
Physical Brute Force / Device Theft

Forced entry, lock bypass, or theft of hardware to extract data offline

indicators
  • CCTV footage of unauthorized entry
  • Badge reader failure/tamper alert
  • Missing hardware in inventory
  • Physical damage to locks/equipment
mitigation
  • Access control vestibule / mantrap
  • Badge + PIN (two-factor physical)
  • CCTV with monitoring
  • Full-disk encryption (FDE) on all devices
  • Cable locks / hardware anchoring
  • Bollards for vehicle-based threats
facility / server room / workstation
vulnerability
RFID Systems Without Encryption

Contactless badges using weak/no encryption can be read and cloned at range

HIGH
attack
RFID Cloning

Attacker uses skimmer near victim to capture badge UID; clones to blank card for facility access

indicators
  • Badge used at unusual time / location
  • Concurrent badge use (same card, different door)
  • Access logs show entry without CCTV match
mitigation
  • Encrypted RFID / smart cards (PKI-based)
  • Multi-factor physical access (badge + PIN)
  • RFID-blocking wallets for personnel
  • Anomaly alerts on concurrent badge use
  • Regular badge audit & revocation
access control system / badge / facility mgmt

Deliberate disruption of cooling, power, or physical environment to cause hardware failure and downtime

mitigation
  • UPS + generators with auto-failover
  • Redundant HVAC systems

◉ Indicators of Malicious Activity — Master Reference
Account Lockout

Brute force or spraying attempts; also insider misuse

Concurrent Sessions

Credential compromise; session token theft; RFID clone in use

Impossible Travel

Login from NYC then London 30min later → credential theft

Resource Consumption

Cryptomining, worm propagation, DoS, ransomware encryption in progress

Resource Inaccessibility

Active DDoS, ransomware lock, hardware/service failure

Blocked Content

WAF/IPS firing; C2 domain blocked by DNS filtering; DLP alert

Out-of-Cycle Logging

Events at 3am; activity during maintenance windows; non-business-hours admin access

Missing Logs

Attacker cleared logs; log tampering; ransomware deleted VSS copies

Published / Documented

CVE published matching asset version; threat intel feed match; dark web leak
