2.5 Mitigation Techniques Used to Secure Enterprise Environment â–¶
Access & Segmentation Controls â–¶
Segmentation
Divide network into zones (VLANs, DMZ, microsegments); limits lateral movement â€” attacker who breaches one segment cannot automatically reach others; critical for ICS/OT environments
Access Control Lists (ACLs)
Explicit permit/deny rules on network devices and file systems; define who/what can communicate with what; most specific rule wins; regular review to remove stale rules
Application Allow List
Only explicitly approved applications permitted to execute; strongest defence against unknown malware and zero-days; high admin overhead; ideal for high-security environments
Isolation
Air-gap critical systems; sandbox suspicious files/processes; isolate compromised hosts during incident response; prevents spread and C2 communication
Least Privilege
Users/processes get minimum access required for their function; reduces blast radius of any single compromise; applies to human accounts, service accounts, and applications
Operational Controls â–¶
Patching
Timely application of vendor security patches; priority based on CVSS score and exploitability; test before deploying to production; patch management tooling (WSUS, SCCM, Ansible)
Encryption
Data at rest (FDE, file/database encryption); data in transit (TLS 1.2/1.3); protects confidentiality even if physical media stolen or network traffic intercepted
Monitoring
Continuous log collection and analysis (SIEM); endpoint monitoring (EDR); network flow analysis; without monitoring, attacks can persist undetected for months (average dwell time: 200+ days)
Configuration Enforcement
Security baselines (CIS Benchmarks); automated drift detection; configuration management tools (Ansible, Chef, Puppet); ensure secure configs persist after changes
Decommissioning
Formally retire and sanitise end-of-life systems; remove from network to eliminate attack surface; data destruction per NIST 800-88 guidelines before disposal
Hardening Techniques â–¶
Encryption at Host Level
Full-disk encryption (BitLocker, FileVault); protects data if device stolen; TPM binding ensures encryption key only released to verified boot environment
Endpoint Protection Installation
AV/EDR deployment; real-time scanning; behavioural detection; threat hunting capability; EDR provides visibility and response (quarantine, kill process, forensic data collection)
Host-Based Firewall
Filters traffic at the individual host; blocks inbound connections to unused ports; complements network firewall â€” defence in depth; configured via GPO in Windows environments
HIPS (Host-Based Intrusion Prevention System)
Monitors and blocks malicious activity on host in real time; detects exploit attempts, privilege escalation, suspicious process behaviour; complements AV with behaviour-based detection
Disabling Unused Ports / Protocols
Close all non-essential network ports; disable protocols not in use (Telnet, FTP, SMBv1); reduce attack surface â€” every open port is a potential entry point
Default Password Changes
First hardening step for any new device; default credentials publicly known (Shodan indexes millions of devices with defaults); change before connecting to network
Removal of Unnecessary Software
Uninstall software not required for function (attack surface reduction); remove bloatware, development tools from production, unused services; minimise installed software inventory
